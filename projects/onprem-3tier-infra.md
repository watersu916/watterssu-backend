## 개요
단일 서버 환경에서는 장애 발생 시 서비스가 중단되고, 스토리지와 애플리케이션이 강하게 결합되어 확장성과 운영 효율이 떨어지는 문제가 있었습니다. 이를 해결하기 위해 DNS, Load Balancer, Web Cluster, NAS(NFS), SAN(NVMe/TCP), Database를 계층별로 분리하여 고가용성과 확장성을 갖춘 Enterprise Linux 기반 3-Tier 인프라를 설계·구축했습니다.

## 프로젝트 범위 및 구성
- **DNS 기반 서비스 접근** - DNS 서버를 직접 구동하여 IP가 아닌 도메인 이름 기반으로 서비스 엔드포인트에 접속할 수 있도록 설계했습니다.
- **Round Robin 기반 트래픽 분산** - L4/L7 로드밸런서(HAProxy)를 다중화하고 라운드로빈 정책을 주입해 유입 트래픽을 각 WAS 노드로 균등 분배했습니다.
- **Active-Active Web Cluster 구성** - 다중 웹 서버 노드가 상시 활성화되어 동작하도록 구성하여 노드 중단 시에도 트래픽 단절이 없는 고가용성을 검증했습니다.
- **공유 스토리지 기반 무중단 콘텐츠 제공** - 분산 노드 간 동일한 소스 파일과 미디어를 바라보도록 NFS 공유 저장소를 마운트해 무중단 운영 상태를 실현했습니다.
- **SAN을 통한 DB Storage 분리** - 블록 스토리지 공유 프로토콜(iSCSI/NVMe/TCP)을 구성하여 DB 데이터 영역을 전용 SAN 볼륨으로 완전 격리했습니다.
- **Storage / Compute Layer 분리 경험** - 연산(Compute) 노드와 저장(Storage) 노드의 생명주기를 완벽히 독립시켜 물리 레이어 수준의 확장 기반을 학습했습니다.
- **Kubernetes Migration을 위한 기반 아키텍처 설계** - 물리 레이아웃 분할 노하우를 바탕으로, 차후 컨테이너 기반 쿠버네티스 환경으로 손쉽게 전환할 수 있도록 구조를 전제했습니다.
